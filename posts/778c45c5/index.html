    多线程的生产者消费者模式实现方式
    发表于 2019-01-16| 更新于 2019-11-05|Java SE

    线程通信

    应用场景：生产者与消费者问题

    • 假设仓库中只能存放一件产品，生产者将生产出来的产品放入仓库，消费者将仓库中产品取走消费；

    • 如果仓库中没有产品，则生产者将产品放入仓库，否则停止生产并等待，知道仓库中的产品被消费者取走为止；

    • 如果仓库中有产品，消费者可以将产品取走消费，否则停止消费并等待，直到仓库中再次放入产品为止；

    分析：这是一个线程同步问题，生产者和消费者共享一个资源，并且生产者和消费者之间相互依赖，互为条件。

    • 对于生产者，没有生产产品之前，要通知消费者等待。而生产了产品之后，又需要马上通知消费者消费；
    • 对于消费者，在消费之后，要通知生产者已经消费结束，需要继续生产新产品以供消费；
    • 在生产者和消费者问题中，不仅需要同步，而且还需要不同线程之间的消息传递。

    解决方式

    管程法

    • 生产者：生产数据的模块（这里模块可能是：方法、对象、线程、进程）；
    • 消费者：处理数据的模块（这里模块可能是：方法、对象、线程、进程）；
    • 缓冲区：消费者不能直接使用生产者的数据，它们之间有个“缓冲区”，生产者将生产好的数据放入“缓冲区”，消费者从“缓冲区”拿要处理的数据。

    信号灯法

    根据一个标识来判断生产者或消费者是否等待，如同交通信号灯，红灯停，绿灯行。

    常用方法

    Java提供了4个方法解决线程之间的通信问题

    方法名 作用
    final void wait() 表示线程一直等待，直到其他线程通知，与sleep不同，会释放锁
    final void wait(long timeout) 指定等待额毫秒数
    final void notifiy() 唤醒一个处于等待状态的线程
    final void notifiyAll() 唤醒同一个对象上所有调用wait()方法的线程，优先级别搞的线程优先调度

    均是java.lang.Object类的方法。
    都只能在同步方法或者同步代码块中使用，否则会抛出异常。

    管程法

    涉及的对象：

    1. 生产者（多线程）
    2. 消费者（多线程）
    3. 缓冲区（并发容器）
    4. 具体数据

    容器需要提供的方法：

    1. 存数据：生产者往容器内添加数据
    2. 取数据：消费者往从容器里拿数据

    存数据和取数据的条件：

    1. 当容器满了，生产者需要等待（wait（））。一旦生产者往容器里添加数据了，就要通知（notify（））消费者消费；
    2. 当容器为空，消费者需要等待（wait（））。一旦消费者往容器里取数据了，就要通知（notify（））生产者生产。
